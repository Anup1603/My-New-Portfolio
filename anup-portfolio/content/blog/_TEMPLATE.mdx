Open with a hook — one or two sentences on why this topic matters and what the
reader will get out of it. Keep it concrete.

## Start with the main idea

Explain one idea per section. Use **bold** for emphasis, _italics_ for nuance,
and [links](https://example.com) for references. Inline `code` looks like this.

- A quick point worth calling out
- Another supporting point

1. Or a numbered step
2. Followed by the next step

> Use a quote block for a key takeaway, a definition, or a cited quote.

### A sub-section that adds detail

Add an image to explain something visually. Put the file in
`public/images/blog/your-post-slug/`, write real alt text, and the quoted text
becomes the caption:

![Descriptive alt text for accessibility](/images/blog/your-post-slug/example.png "Optional caption shown under the image")

## Show code when it helps

Fence code with a language tag to get highlighting, a label, and a copy button:

```ts
export function greet(name: string): string {
  return `Hello, ${name}`;
}
```

## Wrap up with the takeaway

End with the one thing you want the reader to remember. If you filled in
`sources` above, they render automatically as a numbered
"Sources &amp; References" list right below this — no need to add it yourself.
